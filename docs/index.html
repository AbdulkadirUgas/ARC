ARC Split Dashboard

Understand the splits, the weak categories, and the baseline results in one view.

This is the shareable summary view for the ARC analysis work. It focuses on the dataset split taxonomy, the baseline families we tried, and the key conclusion: easy crop and transform families already show some signal, while hard object-compositional categories remain the main weakness and the main target for future synthetic data and solver work.

Main Findings
What the taxonomy and baseline work tell us right now.
1. The dataset is not one uniform problem.
We split ARC into multi-label families instead of forcing one label per task. That lets us separate crop/extract, transform/color-map, tiling, and hard object/compositional reasoning.
2. Prompt-only Qwen is not the path forward.
Qwen improved output formatting and structure in some runs, but exact evaluation accuracy stayed at zero on the tested categories. The bottleneck is rule recovery, not parsing anymore.
3. Easier categories already show some signal.
Deterministic and object-lite baselines show non-zero signal on simpler families like crop and basic transform/color-map tasks. That suggests those categories are already relatively richer and less bottlenecked.
4. Hard object/compositional categories are the weak region.
That is where both the prompt baselines and the current deterministic/compositional baselines break down. It is also the most useful region to target for synthetic data generation and stronger program search.

What This Means Next
Practical recommendation based on the work so far.
Area Current conclusion
Synthetic data Do not generate uniformly across all categories. Target the weak categories, especially hard object/compositional tasks.
LLM prompting Another round of simple prompt-only testing is unlikely to move the benchmark meaningfully.
Next solver direction Move toward stronger program/rule search for object-compositional reasoning, with LLMs used only as proposer or reranker if needed.
Why the taxonomy matters It gives a concrete target map for both synthetic data generation and performance measurement by category rather than a single overall score.
