# PROJECT BLUEPRINT: ARC-FACTORY 

 
**Objective:** Create a procedural generation pipeline capable of producing 1,000,000+ unique, valid ARC-like tasks. This dataset serves as the "Textbook" for teaching core reasoning primitives.

### 1 System Architecture: "Generator of Generators"
We will not ask an LLM to generate grids directly. We will use an LLM to write **Python Programs** that generate grids.

* **The Architect (LLM):** Writes Python scripts based on abstract concepts (e.g., "Gravity").
* **The Worker (CPU):** Executes these scripts with random seeds to produce massive datasets.
* **The Critic (Filter):** Validates that tasks are solvable and non-trivial.

### 2. The Curriculum (Core Primitives)
The generator must cover these 5 fundamental "Chapters" of reasoning:

1.  **Objectness & Topology:**
    * *Concepts:* Connected components, holes, enclosures, contact points, flood fill.
    * *Example Task:* "Identify all blue objects with holes and fill the holes with red."
2.  **Geometry & Symmetry:**
    * *Concepts:* Rotation, reflection, scaling, tiling, cropping, grid expansion.
    * *Example Task:* "Reflect the input pattern across the Y-axis."
3.  **Physics & Causality:**
    * *Concepts:* Gravity, sliding, collision, occlusion (layering), movement paths.
    * *Example Task:* "Move all red pixels 'down' until they hit a green pixel or the edge."
4.  **Arithmetic & Counting:**
    * *Concepts:* Sorting by size/color, majority vote, histograms, sequences.
    * *Example Task:* "Sort the vertical bars from shortest to tallest."
5.  **Abstract Pattern Matching:**
    * *Concepts:* Logical operations (XOR, AND), pattern continuation, analogy.
    * *Example Task:* "Apply the color pattern found in the top-left 3x3 quadrant to all other quadrants."

### 3.2. "The Architect" Prompt Template
You are a Senior Python Engineer specializing in Procedural Generation.

GOAL: Write a robust, self-contained Python script to generate ARC (Abstraction and Reasoning Corpus) tasks for the concept: {CONCEPT_NAME}.

#### REQUIREMENTS:

**Output Format:** The script must return a dictionary with train (3-5 pairs) and test (1 pair) examples. Each grid is a 2D list of integers (0-9).

**Logic:** The transformation rule must be deterministic and logically sound.

**Randomization:** The script must use random heavily. Every execution should produce a visually distinct grid (different sizes, colors, object positions) while maintaining the same logic.

**Constraints:** Handle edge cases (e.g., objects hitting borders, overlapping colors).

### 3.3. "The Critic" (QA) Logic

Before saving a task to the dataset, it must pass these checks:

**Non-Triviality:** Input_Grid != Output_Grid (unless the task is explicitly "Identity").

**Complexity:** The output grid must not be a single solid color (unless the task is "Flood Fill").

**Solvability:** A basic heuristic solver should fail 50% of the time, while a strong solver passes. If a random-forest classifier can solve it instantly, it is too simple.
